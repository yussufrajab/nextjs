## 1. Training Program Overview

### 1.1 Training Objectives

The CSMS training program was designed to achieve the following objectives:

**Primary Objectives:**

1. Enable users to navigate the CSMS interface confidently
2. Train users to perform role-specific tasks independently
3. Ensure understanding of HR request workflows and approval processes
4. Familiarize users with document upload/download procedures
5. Train users on report generation and data analysis
6. Educate users on security policies and best practices
7. Provide hands-on practice with realistic scenarios

**Secondary Objectives:**

1. Build user confidence through supervised practice
2. Identify and address knowledge gaps
3. Gather user feedback for system improvements
4. Establish peer support networks within institutions
5. Create institutional champions for ongoing support

### 1.2 Training Methodology

**Delivery Approach:**

- **Format:** In-person classroom training (primary) + Virtual sessions (for remote participants)
- **Duration:** 2-4 hours per session (role-dependent)
- **Class Size:** 10-15 participants per session for optimal interaction
- **Language:** Bilingual (English and Swahili)
- **Hands-On Practice:** 60% theory, 40% hands-on exercises

**Training Components:**

1. **Presentation:** PowerPoint slides with system walkthroughs
2. **Demonstrations:** Live system demonstrations by trainers
3. **Hands-On Exercises:** Participants perform tasks in test environment
4. **Group Activities:** Collaborative problem-solving scenarios
5. **Q&A Sessions:** Open discussion and clarification
6. **Assessments:** Pre-training and post-training knowledge tests

**Topics Covered:**

- Introduction to CSMS purpose and benefits

- Supported browsers and system requirements

- Login procedure with username and password
- First-time password change

**Hands-On Exercises:**

1. Login with temporary credentials
2. Change password following security policy
3. Navigate main menu
4. Toggle language between English and Swahili
5. View notifications

**Assessment Questions:**

**Outcomes:**
✅ 100% of participants successfully logged in
✅ 97% completed password change on first attempt
✅ 92% correctly answered all assessment questions

### 3.2 Module 2: Role-Specific Workflows

#### 3.2.1 HRO Training (4 hours)

**Topics Covered:**

- Employee profile search and viewing
- Submitting HR requests (all 8 types):
  1. Confirmation requests (probation eligibility check)
  2. Promotion requests (education-based and performance-based)
  3. LWOP requests (duration validation)
  4. Cadre change requests
  5. Retirement requests (compulsory, voluntary, illness)
  6. Resignation requests (3-month vs. 24-hour notice)
  7. Service extension requests
  8. Termination/Dismissal requests
- Document upload requirements (PDF, 2MB limit)
- Request status tracking
- Handling sent-back requests (rectification and resubmission)
- Viewing decision letters and notifications
- Institutional data isolation (only own institution visible)

**Hands-On Exercises:**

1. Search for eligible employee for confirmation
2. Submit confirmation request with document upload
3. Track request status
4. Rectify and resubmit sent-back request
5. View approved request and decision letter

**Scenarios Practiced:**

- Submit confirmation for employee with 13 months probation ✅
- Attempt confirmation for employee with 10 months (validation error) ✅
- Submit promotion with TCU verification for foreign degree ✅
- Submit LWOP with 2-year duration ✅
- Upload multiple PDF documents ✅

**Assessment Results:**

- Average score: 89% (41 HRO participants)
- All participants able to submit at least 3 different request types
- 95% successfully uploaded documents on first try

## 8. Issues and Resolutions

### 8.1 Training Delivery Issues

**Issue #1: Internet Connectivity Interruptions (Week 1)**

- **Description:** Intermittent internet outages during first 2 training sessions
- **Impact:** 15-minute delay in hands-on exercises, participants unable to login
- **Root Cause:** Insufficient bandwidth for 15 concurrent users
- **Resolution:** Upgraded internet bandwidth from 10 Mbps to 50 Mbps
- **Status:** ✅ Resolved - No further connectivity issues

**Issue #2: Training Venue Overcrowding (Week 3, Session 18)**

- **Description:** 21 participants in room designed for 15, uncomfortable
- **Impact:** Limited space for hands-on practice, some participants standing
- **Resolution:** Capped future sessions at 15 participants, added Session 21 for overflow
- **Status:** ✅ Resolved

**Issue #3: Projector Malfunction (Week 2, Session 9)**

- **Description:** Projector failed mid-session, presentations not visible
- **Impact:** 20-minute delay while backup projector set up
- **Resolution:** Backup projector used, primary projector replaced
- **Status:** ✅ Resolved

---

### 8.2 Content-Related Issues

**Issue #4: Swahili Translation Incomplete (Week 1)**

- **Description:** Some system screens and notifications only in English
- **Impact:** Swahili-speaking participants needed English explanations
- **Resolution:** Noted as post-launch enhancement, workarounds provided
- **Status:** ⚠️ Partial - Swahili translations completed post-training

**Issue #5: Advanced Features Not Covered (PO Feedback)**

- **Description:** PO requested advanced reporting features not in training curriculum
- **Impact:** PO felt training was too basic for their needs
- **Resolution:** Additional 2-hour advanced session scheduled and conducted (Session 12)
- **Status:** ✅ Resolved

### 8.3 System Issues Discovered During Training

**Issue #6: Document Upload Timeout for Large Files (Week 1)**

- **Description:** 2MB PDF files timing out during upload from slow connections
- **Impact:** 3 participants unable to complete document upload exercise
- **Resolution:** Increased upload timeout from 30s to 60s, optimized file handling
- **Status:** ✅ Fixed - Deployed during training period

**Issue #7: Report Generation Slow for Large Datasets (Week 2)**

- **Description:** Report with 10,000+ records took >45 seconds to generate
- **Impact:** Participants thought system was frozen
- **Resolution:** Implemented pagination (already in factory test fixes), clarified in training
- **Status:** ✅ Fixed

**Issue #8: AI Complaint Rewriting Inconsistent (Week 3)**

- **Description:** AI rewriting sometimes produced generic responses
- **Impact:** Employees confused about whether to accept AI suggestion
- **Resolution:** Improved Genkit prompts, added "Original vs. AI" comparison view
- **Status:** ✅ Fixed

- **Impact:** Slower pace in hands-on exercises, needed one-on-one assistance
